# Николай — Telegram-бот для рабочей группы

Эта версия бота заточена под один сценарий:

- бот **автоматически отвечает только на поручения руководителя**;
- сейчас руководитель задан как `@denislenivko` и `id=334161944`;
- ответы идут сразу в группу / тему, **без подтверждения от владельца**;
- владельцу бот параллельно шлёт уведомление в личку, чтобы было видно, на что он уже ответил;
- любой участник группы может вызвать команду с анекдотом: `/anekdot`, `/анекдот` или `/joke`.

## Как теперь работает логика

### 1. Бот отвечает только на сообщения руководителя
Бот анализирует сообщения только от заданного руководителя:
- по `user_id`
- и по `username`

По умолчанию:
- `MANAGER_USER_IDS=334161944`
- `MANAGER_USERNAMES=denislenivko`

### 2. На что он реагирует
Бот ловит не только вопросы, а именно **поручения и постановки задач**, например когда в сообщении есть:
- нужно / надо / что делаем / сделать
- дедлайн / сегодня / до 19:00 / срочно
- таблица / реестр / docs.google.com / spreadsheets
- отписываемся / по готовности / дайте статус
- почистить / собрать / создать / обновить / заполнить

### 3. Что он отвечает
Бот не спрашивает у тебя разрешения.
Он сразу отвечает в таком духе:
- «Принял в работу. Я в курсе, подхватываю и вернусь со статусом по готовности.»
- «Ок, взял в работу. Материалы посмотрю, нужные поля заполню и отпишусь по готовности в срок.»
- «Вижу срок, подхватываю сразу. По факту выполнения отпишусь в эту же ветку.»

### 4. Анекдоты для команды
Любой участник группы может написать:
- `/anekdot`
- `/анекдот`
- `/joke`

И бот пришлёт один из 5 коротких непошлых анекдотов.

```env

OWNER_CHAT_ID=твой_личный_chat_id
OWNER_USER_ID=твой_user_id
ALLOWED_CHAT_IDS=-1002383610198
AUTO_REPLY_ENABLED=true
RESPONSE_DELAY_SECONDS=12
TOPIC_COOLDOWN_SECONDS=90
LONG_POLL_TIMEOUT=50
PORT=8080

MANAGER_USERNAMES=denislenivko
```

## Как обновить бота целиком

1. Замени все файлы проекта в GitHub на файлы из нового архива.
2. Закоммить изменения.
3. Дождись автодеплоя Railway.
4. В личке боту отправь `/reload`.
5. Проверь `/status`.

---

## Что важно знать

- Бот **не отвечает на всех подряд** — только на сообщения руководителя.
- На сообщения остальных он не реагирует, кроме команды с анекдотом.
- Если руководитель поменяет username, бот всё равно продолжит работать по `user_id`.
- Если хочешь потом добавить второго руководителя, просто допиши ещё один id в `MANAGER_USER_IDS` через запятую.
