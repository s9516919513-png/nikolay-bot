# Николай — Telegram-бот для рабочей группы

MVP-бот для Telegram-группы / супергруппы с темами.

Что умеет:
- читает новые сообщения в выбранных группах;
- отвечает в той же группе и в той же теме;
- работает по FAQ и ключевым словам;
- в рискованных кейсах дает аккуратный hold-ответ и уведомляет владельца;
- шлет владельцу в личку копию своего ответа;
- умеет ставиться на паузу и включаться обратно.

## Важно
Это **не полноценная ИИ-модель**, а бесплатный боевой каркас:
- без оплаты за LLM;
- ответы берутся из `data/faq.json` и `data/style.json`;
- стиль и знания ты настраиваешь редактированием JSON.

Если позже захочешь, в этот же каркас можно добавить OpenAI / Gemini / другой LLM.

---

## 1) Создай бота в Telegram

Через **@BotFather**:
1. `/newbot`
2. задай имя и username
3. сохрани токен

Потом сделай одно из двух:
- **вариант А:** добавь бота в группу **админом**;
- **вариант Б:** у @BotFather отключи **privacy mode** (`/setprivacy` -> `Disable`) и **заново добавь бота в группу**.

Если бот должен писать тебе уведомления в личку:
1. открой личный чат с ботом;
2. нажми `/start`.

---

## 2) Подготовь проект

Скопируй `.env.example` в `.env` и заполни значения. Приложение само читает `.env` при старте.

### Обязательные переменные

- `TELEGRAM_BOT_TOKEN` — токен от BotFather
- `OWNER_CHAT_ID` — твой chat_id для уведомлений в личку
- `OWNER_USER_ID` — твой user_id, чтобы только ты управлял ботом

### Полезные переменные

- `ALLOWED_CHAT_IDS` — список ID групп через запятую
- `AUTO_REPLY_ENABLED` — включены ли автоответы при старте
- `RESPONSE_DELAY_SECONDS` — сколько бот ждет перед ответом
- `TOPIC_COOLDOWN_SECONDS` — защита от спама в одной теме

---

## 3) Как узнать свой chat_id и chat_id группы

### Свой chat_id
1. Запусти бота локально или на Railway
2. Напиши боту `/start`
3. Временно добавь в `app.py` логирование апдейтов **или** воспользуйся любым Telegram userinfo/debug ботом

### chat_id группы
Самый простой путь:
1. добавь Николая в группу;
2. временно оставь `ALLOWED_CHAT_IDS` пустым;
3. посмотри логи Railway — там будут видны chat_id, если добавить лог на входящие сообщения.

Для первого запуска можно просто оставить `ALLOWED_CHAT_IDS` пустым, проверить работу и потом уже зафиксировать нужные chat_id.

## 4) Локальный запуск

```bash
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
python app.py
```

Проверка health endpoint:

```bash
curl http://localhost:8080/
```

---

## 5) Деплой на Railway через GitHub

1. Создай новый репозиторий на GitHub.
2. Залей в него все файлы проекта.
3. На Railway нажми **New Project** -> **Deploy from GitHub repo**.
4. Выбери репозиторий.
5. В разделе **Variables** добавь переменные из `.env.example`.
6. Railway подхватит `railway.json` и стартанет сервис командой `python app.py`.

Если Railway не подхватил стартовую команду, задай вручную:

```bash
python app.py
```

## 6) Как отредактировать стиль под себя

### `data/faq.json`
Тут лежат темы и готовые ответы.

Структура элемента:

```json
{
  "id": "price",
  "priority": 2,
  "keywords": ["цена", "стоимость", "сколько стоит"],
  "response_variants": [
    "Сориентирую по условиям...",
    "По стоимости подскажу предметно..."
  ]
}
```

### `data/style.json`
Тут лежат:
- слова, которые считаются вопросом;
- риск-слова;
- шаблоны fallback-ответов;
- шаблоны hold-ответов.

---

## 7) Команды владельца

Напиши боту в личку:

- `/start` — регистрация личного чата владельца
- `/status` — статус бота
- `/pause` — поставить автоответы на паузу
- `/resume` — снова включить автоответы
- `/reload` — перечитать FAQ и стиль из JSON
- `/test` — тестовое сообщение

---

## 8) Как работает логика

1. Бот получает сообщение в группе.
2. Если это похоже на вопрос — ставит его в очередь.
3. Ждет `RESPONSE_DELAY_SECONDS`.
4. Если в теме уже появилась активность от другого человека — пропускает ответ.
5. Если вопрос типовой — отвечает по FAQ.
6. Если вопрос рискованный или непонятный — дает аккуратный промежуточный ответ и шлет тебе уведомление.

---

## 9) Что стоит улучшить следующим этапом

Когда MVP заработает, рекомендую добавить:
- SQLite/Postgres для истории;
- белые/черные списки тем;
- отдельные правила для VIP-клиентов;
- подключение реальной LLM;
- веб-панель для редактирования FAQ без правки JSON.

---

## 10) Ограничения текущей версии

- не обучается автоматически на твоих переписках;
- не понимает смысл как настоящая LLM;
- не хранит устойчивую историю диалогов в БД;
- может требовать тонкой настройки FAQ, чтобы попадать в твой стиль.

Но как бесплатный первый этап — это уже рабочий каркас.
